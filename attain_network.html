ATTAIN Network¶

Convenience class used to handle a network in the context of using MCL for clustering and posterior analysis.

add_gene_weights(file)[source]¶

Add weights information to the _genes DataFrame

From the provided list of weights, the selected values are added to the internal _genes list.

Processing of weights: All values are made positive (by taking the abs) of each, and missing values are replaced by the median of the weights so as to avoid blanks.

Since the nature of the clustering algorithm requires the weights to be to be associated to the edges of the network, once the information is loaded to the _genes, a call to update the information available in the _edges dataframe is also made.

Parameters

file (str) – The file that contains all weights information

init_edges_weights()[source]¶

Give each _edge a weight, based on the current weights of genes

MCL is based in the idea that the weight (or strengh) of the edges between nodes in a network influences the way in which the nodes should be clustered together, thus it requires the edges to have an associated weight.

This method initialize the weight of each edge as the sum of the weights of its source and target nodes, and adds this informatation as a new column to the _edges DataFrame.

init_gene_edge_mapping()[source]¶

Map the _edges identifiers to _genes index

Genes and Edges are mainly identified through a GENEID, an identifier for biological data that is not correlative for the current network.

To use MCL’s algorithm, we need the nodes and edges in the network to use a correlative identifier, that goes from 0 to n-1, where n is the number of nodes in the network. Such identifier is easily achieved by selecting the index of the _genes DataFrame.

This method creates two new columns on the _edges dataframe that use the gene index to encode the edge’s source and end points.

init_matrix(unweighted=True)[source]¶

Initialize the sparse matrix representation of the network

In order to run the MCL algorithm a squared, undirected (two-way link) sparse matrix representation of the network is required. We define this matrix based on the information stored in the _edges dataframe.

shuffle_node_weights()[source]¶

Shuffle the nodes selected weight

Shuffling the gene’s weights implies a change in the weight associated to each edge in the network, thus, everytime this function is invoked, we need to both, shuffle the weights in the _genes dataframe, and then recalculate the corresponding weights in the _edges dataframe.

Returns

True if the weights were shuffled. False in any other situation

to_csv()[source]¶

Save the current network and clustering results to csv files

In order to persist calculations and use them later for processing, the _genes and _clusters DFs can be saved to csv files. The generated files are placed by default under a ‘results’ folder in the current directory. Names for the files need to be individually indicated by the user on execution.
